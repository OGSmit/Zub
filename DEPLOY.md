# Деплой на GitHub Pages

## Настройка репозитория

1. Создайте репозиторий на GitHub (если еще не создан)
2. Убедитесь, что основная ветка называется `main`

## Настройка baseURL

В файле `nuxt.config.ts` настройте `baseURL` в зависимости от имени вашего репозитория:

```typescript
app: {
  baseURL: process.env.NODE_ENV === 'production' ? '/ИМЯ_РЕПОЗИТОРИЯ/' : '/',
  // ...
}
```

**Важно:** 
- Если репозиторий называется `Zub`, то baseURL должен быть `/Zub/`
- Если репозиторий называется `username.github.io`, то baseURL должен быть `/`
- Замените `ИМЯ_РЕПОЗИТОРИЯ` на актуальное имя вашего репозитория

## Автоматический деплой

Проект настроен на автоматический деплой через GitHub Actions:

1. При каждом push в ветку `main` автоматически запускается workflow
2. Проект собирается в статический сайт
3. Результат деплоится в ветку `gh-pages`
4. Сайт становится доступным по адресу: `https://ВАШ_USERNAME.github.io/ИМЯ_РЕПОЗИТОРИЯ/`

## Включение GitHub Pages

1. Перейдите в Settings вашего репозитория
2. Откройте раздел Pages
3. В Source выберите "Deploy from a branch"
4. Выберите ветку `gh-pages` и папку `/ (root)`
5. Нажмите Save

## Ручной деплой

Если нужно задеплоить вручную:

```bash
npm run generate
```

Затем загрузите содержимое папки `.output/public` в ветку `gh-pages`.

## Проверка деплоя

После деплоя сайт будет доступен по адресу:
- `https://ВАШ_USERNAME.github.io/ИМЯ_РЕПОЗИТОРИЯ/`

Время деплоя обычно занимает 1-2 минуты.
